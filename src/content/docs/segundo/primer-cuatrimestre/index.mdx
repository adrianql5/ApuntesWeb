## Opiniones y Consejos

### POO
- Si la asignatura se llamase *Monopoly* sería mucho más preciso.
- **Id a las teóricas**.
- Programa en Java sin saber programar en Java define la asignatura.
- El proyecto está guapo pero quita muchas horas.
- El examen práctico se resume en usar HashMaps; la parte teórica requiere saber absolutamente todo sobre Java.
- En el final no solo tienes que aprobar un examen, sino **3 exámenes en uno** (2 partes de teoría + 1 práctica).

### Sistemas Operativos I
- **Asignatura más difícil hasta la fecha**.
- No ir a teóricas.
- Las presentaciones son malas, el Tanenbaum también. El Carretero es mejor opción.
- El final no es muy difícil; la asignatura ya de por sí es complicada.
- El profesor no da notas detalladas: solo *mal*, *regular* o *bien* por pregunta.

### AED
- Un poco más difícil que Programación I.
- Las prácticas quitan tiempo pero están bien.
- El examen final es un cachondeo, lleva repitiéndolo desde 2017.
- Es más fácil suspender uno de los tests del curso que el final.
- **Si te la estudias el día antes por la tarde, la apruebas**.

### Redes
- **Segunda asignatura más difícil hasta la fecha**.
- La materia está muy bien pero pierdes tiempo intentando entender cosas (Control de Congestión, Subredes). Renta mirar vídeos de YouTube (Naseros sobre todo).
- La información que dan es escasa; tienes que buscarte la vida.
- Mirarse las chorradas de teoría (IPsec, DHCP, BGP) porque las preguntan en el final.
- **Id a teóricas**.

### Bases de Datos
- Tiene mucha carga de trabajo aburridísima.
- El examen final se divide en: teoría (aburrida), hacer un MER, y SQL.
- MER y SQL son interesantes; el resto sobra.
- El SQL del final suele ser bastante difícil.
